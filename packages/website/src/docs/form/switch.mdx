# Switch

A `<Switch>` is essentially a customized checkbox with a toggleable interface to switch between an enable and disabled state.

## Import

```jsx
import { Switch, SwitchField } from 'bumbag'
```

## Usage

```jsx-live
<Switch label="Check me" />
```

### Fields

You can also use the `<SwitchField>` component, which internally has a [FieldWrapper](/form/field-wrapper) to wrap form controls such as labels, hint text and validation text.

It accepts a combination of [Switch props](/form/switch/#props) and [FieldWrapper props](/form/field-wrapper/#props).

```jsx-live
<SwitchField
  switchLabel="Check me"
  label="Check below to subscribe"
/>
```

### Disabled

Make the switch disabled with the `disabled` prop.

```jsx-live
<Switch disabled label="Click me" />
<Switch checked disabled label="Click me" />
```

### Colors

Add colors to your switch by using the `palette` prop.

```jsx-live
<Switch defaultChecked palette="secondary" label="Check me" />
<Switch defaultChecked palette="success" label="Check me" />
<Switch defaultChecked palette="danger" label="Check me" />
<Switch defaultChecked palette="warning" label="Check me" />
```

### States

A switch can use different states (as per palette) such as `danger`, `success` and `warning`.

```jsx-live
<Switch state="danger" label="Click me" />
<Switch state="success" label="Click me" />
<Switch state="warning" label="Click me" />
<Switch state="primary" label="Click me" />
```

### Controlled Usage

The previous examples are examples of uncontrolled usage. To control the value yourself, use a combination of `onChange` and `value`.

Note: Do not use `defaultValue` with controlled inputs!

```function-live
function Example() {
  const [checked, setChecked] = React.useState(true);

  return (
    <Switch
      label="Click me"
      checked={checked}
      onChange={e => setChecked(e.target.checked)}
    />
  );
}
```

## Accessibility

The `<Switch>` component follows the [WAI ARIA Checkbox Pattern](https://www.w3.org/TR/wai-aria-practices/#checkbox).

### Rules

- The switch must have a `label` prop to identify it's label. If a `label` prop is not provided, an `aria-label` or `aria-labelledby` prop can be specified.

#### Patterns

- The switch has a role of `checkbox`.
- The switch has an accessible label specified via the `label`, `aria-labelledby` or `aria-label` prop.
- When the switch has focus, pressing the <Key>Space</Key> key changes the state of the checkbox.

#### References

- [WAI ARIA Checkbox Pattern](https://www.w3.org/TR/wai-aria-practices/#checkbox)
