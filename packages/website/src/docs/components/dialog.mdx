# Dialog

## Import

```jsx
import { Dialog, Modal } from 'bumbag';
```

## Usage

The `<Dialog.Modal>` component renders a dialog window that overlays the screen & disables interaction with the background. It inherits the [Modal component](/utilities/modal).

It is recommended to use `<Dialog.Modal>` with the `<Modal>` utilities like `<Modal.State>` & `<Modal.Disclosure>`. These components will help you manage the modal state & accessibility attributes on the elements.

```jsx-live
<Modal.State>
  <Dialog.Modal title="This is a title">
    Lorem ipsum dolor sit amet, consectetur adipiscing elit. Suspendisse diam ipsum, cursus id placerat congue, ultrices
    eget lectus. Duis posuere, lacus sed tristique commodo, sapien turpis mollis nunc, vestibulum consectetur lectus
    augue sit amet justo.
  </Dialog.Modal>
  <Modal.Disclosure use={Button}>
    Open modal
  </Modal.Disclosure>
</Modal.State>
```

### Action buttons

It is common to have action buttons in a dialog's footer. Use the `showActionButtons` prop to enable them. Dialog also accepts the `actionButtonsProps` that accepts the props of the [`<ActionButtons>` component](/components/button#actionbuttons-props).

```jsx-live
<Modal.State>
  <Dialog.Modal showActionButtons title="This is a title">
    Lorem ipsum dolor sit amet, consectetur adipiscing elit. Suspendisse diam ipsum, cursus id placerat congue, ultrices
    eget lectus. Duis posuere, lacus sed tristique commodo, sapien turpis mollis nunc, vestibulum consectetur lectus
    augue sit amet justo.
  </Dialog.Modal>
  <Modal.Disclosure use={Button}>
    Open modal
  </Modal.Disclosure>
</Modal.State>
```

```jsx-live
<Modal.State>
```

## Accessibility

The `<Dialog.Modal>` component follows the [WAI ARIA Dialog (Modal) Pattern](https://www.w3.org/TR/wai-aria-practices/#dialog_modal).

### Rules

- A `Dialog.Modal` must be triggered via `<Modal.Disclosure>`, or `Modal.Disclosure.useProps`.

#### Patterns

- If a dialog has a **title**, then the `aria-labelledby` attribute is set on the component.
- If a dialog has a **description**, then the `aria-describedby` attribute is set on the component.
- `Dialog.Modal` has a role of `dialog`.
- `Dialog.Modal` has the `aria-modal` attribute set to `true`, unless the `modal` prop is set to `false`.
- When the modal opens, focus is set to the first focusable element in the modal.
- Focus is trapped inside the modal, unless the `modal` prop is set to false.
- <Key>Esc</Key> closes the modal unless `hideOnEsc` is `false`.
- Clicking outside the modal closes it unless `hideOnClickOutside` is `false`.
- When the modal closes, focus returns to the disclosure that triggered it.

#### References

- [WAI ARIA Dialog (Modal) Pattern](https://www.w3.org/TR/wai-aria-practices/#dialog_modal)
